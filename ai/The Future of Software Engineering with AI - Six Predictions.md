## AI 요약

### 1. 아티클 개요

Pragmatic Summit(샌프란시스코, 500명 참석)과 "The Future of Software Development" 워크숍(유타 디어밸리, ~50명 참석)에서 얻은 인사이트를 종합한 아티클이다. Martin Fowler, Kent Beck, Steve Yegge, Gene Kim 등 소프트웨어 공학 거장들이 참여했으며, 2026년 현재 AI가 소프트웨어 엔지니어링에 미치는 실질적 변화를 6가지 예측으로 정리한다.

| 항목 | 내용 |
|------|------|
| 핵심 주제 | AI가 소프트웨어 엔지니어링 조직, 팀 구성, 개발 방식에 미치는 구조적 변화 |
| 주요 이벤트 | Pragmatic Summit (SF), Future of SW Dev Workshop (Deer Valley) |
| 핵심 참여자 | Martin Fowler, Kent Beck, Steve Yegge, Gene Kim, Laura Tacho, Thomas Dohmke, Rajeev Rajan |
| 핵심 메시지 | AI는 증폭기(Amplifier)다 — 건강한 조직은 더 강해지고, 역기능 조직은 더 무너진다 |
| 시점 | 2026년 2월 (Agile Manifesto 25주년 시점) |

### 2. 예측 1: 데이터 vs 과대광고 — AI로 승리하는 조직의 조건

Laura Tacho(前 DX CTO)의 키노트에서 공개된 독점 데이터가 핵심이다. AI 도구 채택 자체보다 **조직의 기존 건강도**가 결과를 결정한다.

| 지표 | 수치 |
|------|------|
| 개발자 AI 코딩 도구 월간 사용률 | 92% |
| AI로 절약하는 주당 시간 (자기 보고) | ~4시간 |
| 건강한 조직의 고객 대면 인시던트 변화 | **50% 감소** |
| 역기능 조직의 인시던트 변화 | **2배 증가** |

```
┌──────────────────────────────────────────────────────────────────┐
│                AI는 증폭기(Amplifier)다                            │
│                                                                  │
│   건강한 조직                         역기능 조직                  │
│   ┌──────────────────┐               ┌──────────────────┐        │
│   │ 명확한 프로세스    │               │ 혼란스러운 프로세스│        │
│   │ 강한 엔지니어링    │               │ 기술 부채 방치    │        │
│   │ 문화              │               │ 사일로화된 팀     │        │
│   └────────┬─────────┘               └────────┬─────────┘        │
│            │ + AI                              │ + AI             │
│            ▼                                   ▼                  │
│   ┌──────────────────┐               ┌──────────────────┐        │
│   │ 인시던트 50% 감소 │               │ 인시던트 2배 증가 │        │
│   │ 생산성 극대화     │               │ 혼란 가속화       │        │
│   │ 팀 역량 증폭      │               │ 문제 은폐 → 폭발  │        │
│   └──────────────────┘               └──────────────────┘        │
│                                                                  │
│   "AI is an accelerator, it's a multiplier, and it is moving     │
│    organizations in different directions." — Laura Tacho          │
└──────────────────────────────────────────────────────────────────┘
```

Tacho는 현재 상황을 **우주 경쟁(Space Race)**과 **대항해 시대(Age of Exploration)**에 비유했다. 경이로움과 회의주의의 균형이 필요하며, 가장 높은 야망은 달성하지 못하더라도 의미 있는 진전을 이루고 있다는 것이다.

> "Stay grounded, stay skeptical, stay human. Most of all, stay pragmatic." — Laura Tacho

### 3. 예측 2: AI 네이티브 엔지니어링 조직의 등장

Thomas Dohmke(Entire.io CEO, 前 GitHub CEO)와 Rajeev Rajan(Atlassian CTO)의 대담에서 도출된 핵심 변화들이다.

```
┌──────────────────────────────────────────────────────────────────┐
│            AI 네이티브 팀의 4가지 특성 변화                         │

│   ① 제로 코드 팀 등장                                             │
│   ┌──────────────────────────────────────────────┐               │

│                                                                  │

│   ┌──────────────────────────────────────────────┐               │
│   │ 에이전트 = "스파링 파트너"                       │               │
│   │ 코드 리뷰 에이전트, 브레인스토밍 에이전트,       │               │
│   │ 리서치 에이전트로 외로움 해소                    │               │
│   └──────────────────────────────────────────────┘               │
│                                                                  │
│   ③ 기업 IT 정책이 경쟁 열위가 되는 시대                           │
│   ┌──────────────────────────────────────────────┐               │
│   │ Atlassian CTO: 제한적 IT 정책 우회 위해         │               │

│   │ → 스타트업이 대기업 대비 우위를 유지하는 이유    │               │

│                                                                  │

│   │ 밤에 에이전트에 작업 할당 → 아침에 결과 리뷰     │               │

```

> "Efficiency framing is missing the point. It's more about what you can create now with AI which you could not before." — Thomas Dohmke

### 4. 예측 3: 미드레벨 엔지니어의 조용한 위기

주니어 개발자 위기에 이어, 이제 **미드레벨 엔지니어**가 새로운 위기에 직면하고 있다.

```
┌──────────────────────────────────────────────────────────────────┐
│              경력 단계별 AI 시대 위치                               │
│                                                                  │
│   주니어 (0~2년)          미드레벨 (3~7년)        시니어 (8년+)    │
│   ┌──────────────┐       ┌──────────────┐       ┌──────────────┐│
│   │ AI 도구와 함께│       │ "낀 세대"     │       │ 깊은 도메인   ││
│   │ 성장한 세대   │       │              │       │ 지식 보유     ││
│   │              │       │              │       │              ││
│   │ AI 네이티브로 │       │ AI 이전 방식에│       │ AI가 대체     ││
│   │ 높은 생산성   │       │ 익숙하지만    │       │ 못하는 경험적 ││
│   │              │       │ AI 활용에서   │       │ 판단력       ││
│   │              │       │ 뒤처질 위험   │       │              ││
│   └──────────────┘       └──────────────┘       └──────────────┘│
│        ▲ 강점                ▲ 위기                ▲ 강점        │
│   AI 도구 친숙도 높음    양쪽 다 애매한 위치    아키텍처/판단력    │
└──────────────────────────────────────────────────────────────────┘
```

신입은 AI 도구와 함께 성장하여 높은 생산성을 보이고, 시니어는 AI가 대체할 수 없는 경험과 판단력을 가지고 있다. 그 사이에 낀 미드레벨 엔지니어가 "조용한 위기(Quiet Crisis)"에 처해 있다.

### 5. 예측 4~5: AI 네이티브 시대의 리팩토링과 애자일의 미래

#### 리팩토링의 재정의
AI가 대부분의 코드를 생성하는 시대에도 리팩토링은 **여전히 중요**하다. 오히려 AI 생성 코드의 품질 관리와 구조적 정리가 더 중요해진다.

#### 애자일의 진화 (25주년)
디어밸리 워크숍은 Agile Manifesto 25주년을 기념하는 자리이기도 했다. 주목할 변화는 **XP(Extreme Programming)의 귀환**이다. 애자일 이전의 실천법들이 AI 시대에 다시 주목받고 있다.

### 6. 핵심 관찰: 전례 없는 동시다발적 AI 채택

과거 기술 전환과 달리, AI 채택은 **모든 유형의 조직에서 동시에** 일어나고 있다.

```
┌──────────────────────────────────────────────────────────────────┐
│         기술 전환의 채택 속도 비교                                  │
│                                                                  │
│   과거 기술 전환 (모바일, 클라우드, 크립토)                         │
│   ┌──────────────────────────────────────────────────────┐       │
│   │ 인디 개발자 → 스타트업 → 대형 테크 → 전통 기업         │       │
│   │   [1단계]     [2단계]    [3단계]     [4단계]           │       │
│   │   (수년에 걸친 순차적 채택)                             │       │
│   └──────────────────────────────────────────────────────┘       │
│                                                                  │
│   AI 전환 (2024~2026)                                            │
│   ┌──────────────────────────────────────────────────────┐       │
│   │ 인디 개발자 + 스타트업 + 대형 테크 + 전통 기업          │       │
│   │            [동시 채택 — 뒤처진 곳 없음]                 │       │
│   └──────────────────────────────────────────────────────┘       │
│                                                                  │
│   구체적 사례:                                                    │
│   • John Deere (200년 역사 농기계): 3~4인 팀으로 축소            │
│   • Wealthsimple (캐나다 핀테크): Claude Code 전사 도입          │
│   • 10,000+ 개발자 대기업: AI 플랫폼 팀 신설, 외주 제거 목표     │
│   • 대형 금융기관: CTO 주도 에이전트 전사 도입                    │
└──────────────────────────────────────────────────────────────────┘
```

### 7. 팀 규모의 구조적 변화

여러 엔지니어링 리더들이 공통적으로 보고하는 팀 규모 축소 추세이다.

| 구분 | 이전 (Two-Pizza Team) | 이후 (One-Pizza Team) |
|------|----------------------|----------------------|
| 팀 규모 | 6~10명 | 3~4명 |
| 생산성 배수 | 1x (기준) | 2~5x |
| 적용 범위 | 스타트업 중심 | 200년 역사 농기계 회사까지 |
| 핵심 도구 | Claude Code, Copilot, Cursor, Devin 등 |

### 8. 디어밸리 선언문

Agile Manifesto 서명 25년 후 같은 유타 주에서, Kent Beck, Laura Tacho, Steve Yegge가 공동 작성한 선언문이다.

> "Organizations are constrained by human and systems-level problems. We remain skeptical of the promise of any technology to improve organizational performance without first addressing human and systems-level constraints. We remain skeptical and we remain human."
> — Kent Beck, Laura Tacho, Steve Yegge

이 선언문의 핵심은 **기술 만능주의에 대한 경계**다. AI가 아무리 강력해도, 조직의 인간적·시스템적 제약을 먼저 해결하지 않으면 기술만으로 성과를 개선할 수 없다는 입장이다.

### 9. 유일한 반론: 임베디드 시스템 엔지니어

Pragmatic Summit에서 만난 유일한 "AI 에이전트에 굴복하지 않은" 참석자는 Assembly와 C 코드를 작성하는 임베디드 시스템 엔지니어였다.

| 항목 | 내용 |
|------|------|
| AI 코드 생성 비율 | 1/3 ~ 1/2 (Opus 4.5 출시 이후) |
| 에이전트 사용 여부 | 사용하지 않음 (유일한 회의론자) |
| 추세 | AI 비율이 계속 상승 중 |
| 시사점 | 저수준 프로그래밍에서도 AI 영향력이 확대되고 있음 |

## 내가 얻은 인사이트

### 조직 역량 관점
1. **AI는 리트머스 시험지다**
   - AI가 "건강한 조직은 50% 인시던트 감소, 역기능 조직은 2배 증가"라는 데이터는 강력하다. 이는 AI 도입 자체가 목적이 아니라, AI 도입 전에 조직의 기본기(프로세스, 문화, 기술 부채 관리)를 점검하는 것이 우선이라는 것을 수치로 증명한다. AI 도입 ROI를 측정하기 전에, 먼저 "우리 조직은 AI를 통해 증폭될 만한 가치가 있는가?"를 물어야 한다.

2. **기업 IT 정책의 역설적 효과**
   - Atlassian CTO가 자사 IT 정책을 우회하기 위해 개인 노트북을 산 일화는 상징적이다. 보안과 통제를 위한 IT 정책이 오히려 혁신의 병목이 되고, 가장 동기부여된 직원들이 "그림자 IT(Shadow IT)"로 우회하는 구조는 새로운 현상이 아니지만, AI 시대에 그 비용이 기하급수적으로 커지고 있다.

### 팀 구성 변화 관점
3. **"원-피자 팀"의 보편화와 그 의미**
   - Two-Pizza Team(6~10명)에서 One-Pizza Team(3~4명)으로의 전환은 단순한 인원 감축이 아니다. 이는 팀 내 커뮤니케이션 오버헤드가 극적으로 줄어들면서, 더 빠른 의사결정과 높은 오너십이 가능해진다는 구조적 변화다. 다만 이 변화는 각 팀원의 역량 기준이 훨씬 높아진다는 것을 의미하기도 한다.

4. **미드레벨 엔지니어의 "샌드위치 위기"**
   - 주니어의 AI 네이티브 생산성과 시니어의 대체 불가 경험 사이에 낀 미드레벨의 위기는 설득력 있다. 특히 미드레벨 엔지니어가 "AI 이전 방식에 익숙하지만, AI 활용에서 뒤처질 위험"에 처한다는 분석은, 경력 개발에서 "AI 리터러시"가 단순 옵션이 아닌 생존 조건이 되었음을 보여준다.

### 기술 채택 패턴 관점
5. **동시다발적 채택이라는 전례 없는 현상**
   - 모바일, 클라우드, 크립토가 수년에 걸쳐 순차적으로 확산된 것과 달리, AI는 John Deere부터 핀테크 스타트업까지 동시에 채택되고 있다는 관찰은 중요하다. 이는 "남들 먼저 해보고 따라가면 된다"는 기존 전략이 더 이상 통하지 않음을 의미한다. 느린 채택자(Late Adopter)에게 학습할 시간적 여유가 없는 것이다.

6. **디어밸리 선언문이 던지는 본질적 질문**
   - "기술이 아닌 인간과 시스템의 제약이 조직을 제한한다"는 선언은, AI 시대의 핵심 아이러니를 짚는다. 가장 강력한 기술 도구를 가졌지만, 그것을 제대로 활용하지 못하게 막는 것은 결국 인간의 조직 운영 방식이라는 것이다. Agile Manifesto가 25년 전에 "프로세스보다 사람을"이라고 말했던 것과 같은 맥락이되, AI라는 훨씬 강력한 도구 앞에서 그 메시지가 더욱 절실해졌다.
