## AI 요약

### 1. LCM(Large Concept Models)이란?

LCM은 Meta AI가 2024년 12월 공개한 **문장(개념) 단위로 동작하는 언어 모델 아키텍처**이다. 기존 LLM이 토큰(subword) 단위로 다음 토큰을 예측하는 것과 달리, LCM은 "개념(concept) = 한 문장" 단위로 다음 문장의 **의미 임베딩**을 예측한다. 인간이 단어 하나하나가 아니라 아이디어·문장 단위로 사고하듯, 더 높은 수준의 추상 공간에서 추론하는 것을 목표로 한다.

| 특성 | 설명 |
|------|------|
| 동작 단위 | 문장(Sentence = Concept) |
| 표현 공간 | SONAR 임베딩 공간 (언어/모달리티 독립) |
| 지원 언어 | 텍스트 200개 언어, 음성 입력 76개 언어 |
| 예측 대상 | 다음 문장의 임베딩 벡터 |
| 학습 방식 | MSE 회귀 / Diffusion / Quantization |
| 모델 규모 | 1.6B (탐색) → 7B (스케일업), 학습 데이터 약 2.7T 토큰 |
| 시퀀스 압축 | 1,000 토큰 문서 ≈ 50 개념 (약 20배 축소) |

---

### 2. 왜 토큰이 아닌 "개념"인가?

기존 LLM의 한계는 다음과 같다.

```
[토큰 단위 LLM의 문제점]
 ├─ 긴 시퀀스에서 attention 비용 O(N²) 폭증
 ├─ 언어마다 토크나이저가 달라 다국어 일반화 어려움
 ├─ 멀티모달(음성/비디오) 통합이 부자연스러움
 └─ 계층적·추상적 계획(plan) 수립에 취약
```

LCM은 이를 **"언어·모달리티와 무관한 의미 공간에서 직접 추론"** 하는 방식으로 해결한다.

```
 기존 LLM:   [tok1][tok2][tok3]...[tok1000]    → 다음 토큰 예측
              └───────── 수천 스텝 ─────────┘

 LCM:        [Concept1][Concept2]...[Concept50]  → 다음 개념(문장) 예측
              └──── 20배 짧은 시퀀스 ────┘
```

---

### 3. SONAR 임베딩 공간

LCM의 기반이 되는 표현 공간은 Meta가 이전에 공개한 **SONAR**이다.

| 항목 | 내용 |
|------|------|
| 공간 성격 | 언어·모달리티 독립적 문장 임베딩 |
| 학습 목표 | 번역(MT) + Denoising AutoEncoding + MSE 정렬 |
| 텍스트 | 200개 언어 인코딩/디코딩 |
| 음성 | 76개 언어 입력 |
| 활용 방식 | 입력 문장 → SONAR 인코더 → 임베딩 / 출력 임베딩 → SONAR 디코더 → 문장 |

LCM 학습 시 SONAR 인코더/디코더는 **동결(frozen)** 되어 있다. 모델은 오직 임베딩 공간 위에서 다음 임베딩을 만들어낼 뿐이다.

---

### 4. Base-LCM 아키텍처

가장 단순한 변형으로, MSE 회귀로 학습한다.

```
 [문장1] [문장2] [문장3]
    │      │      │
    ▼  SONAR Encoder (frozen)
 [e1]   [e2]   [e3]
    │      │      │
    ▼  PreNet (정규화 + Transformer 차원으로 매핑)
 ┌──────────────────────────┐
 │   Transformer Decoder    │   ← 다음 개념 임베딩 예측
 └──────────────────────────┘
    │
    ▼  PostNet (SONAR 차원으로 역투영 + denormalize)
 [ê4] ──▶ SONAR Decoder (frozen) ──▶ "문장4"
```

- **학습 목표**: `MSE(ê_{t+1}, e_{t+1})`
- **약점**: 문맥이 주어졌을 때 가능한 다음 문장은 여러 개인데, MSE는 이 모든 가능성의 **평균** 임베딩으로 수렴 → 밍밍하고 확률적으로 부정확한 결과

---

### 5. Diffusion 기반 LCM

Base-LCM의 "다중 모드 문제"를 해결하기 위해 **조건부 Diffusion**을 도입한다. 노이즈를 점진적으로 제거하며 다음 개념 임베딩을 생성한다.

#### 5-1. One-Tower

```
입력: [e1, e2, e3, e4 + noise(t)]                    timestep t
       └────── clean context ─────┘└─ noisy target ─┘
                        │
                        ▼
             ┌───────────────────┐
             │ Single Transformer│  (context + denoising 동시 수행)
             └───────────────────┘
                        │
                        ▼  반복 (t → t-1 → ... → 0)
                  denoised  ê4
```

#### 5-2. Two-Tower (최고 성능)

맥락 인코딩과 디노이징을 **분리**한다.

```
 ┌─ Tower 1: Contextualizer ─┐
 │ [e1, e2, e3] → context h  │
 └───────────────┬───────────┘
                 │ cross-attention
                 ▼
 ┌─ Tower 2: Denoiser ────────────────────┐
 │  e4 + noise(t)  ──▶ reverse diffusion  │
 │                       ├─ step T        │
 │                       ├─ step T-1      │
 │                       └─ step 0 → ê4   │
 └────────────────────────────────────────┘
```

- Contextualizer는 **한 번만** 실행, Denoiser는 **T번** 반복
- 추론 효율과 성능을 동시에 잡은 구조

---

### 6. Quant-LCM

연속 임베딩을 직접 다루는 대신, **Residual Vector Quantization(RVQ)** 로 이산 코드북으로 변환한다.

```
 연속 SONAR 벡터 e
       │
       ▼  RVQ (계층적 양자화)
 [code_1, code_2, ..., code_k]   ← 이산 토큰 시퀀스
       │
       ▼  Transformer (cross-entropy 학습)
 다음 코드 예측
```

- **장점**: 메모리 효율, LLM과 같은 이산 토큰 학습 파이프라인 재사용 가능
- **단점**: 양자화 손실, 표현력 제약

---

### 7. 변형 모델 비교

| 변형 | 학습 방식 | 장점 | 단점 |
|------|-----------|------|------|
| Base-LCM | MSE 회귀 | 단순, 빠른 추론 | 평균 임베딩으로 수렴, 다양성 부족 |
| One-Tower Diffusion | Denoising | 다중 모드 표현 가능 | 맥락 재계산 비효율 |
| Two-Tower Diffusion | Denoising (분리) | **최고 요약 품질**, 효율적 | 구현 복잡 |
| Quant-LCM | Cross-Entropy on RVQ | 이산 파이프라인 재사용 | 양자화 손실 |

---

### 8. 추론 흐름

```
 사용자 문서 (긴 텍스트)
        │
        ▼ 문장 분리
 [문장1][문장2]...[문장N]
        │
        ▼ SONAR Encoder (frozen)
 [e1][e2]...[eN]
        │
        ▼ LCM (next-concept 예측, autoregressive)
 [ê_{N+1}][ê_{N+2}]...
        │
        ▼ SONAR Decoder (frozen, 원하는 언어로)
 생성된 문장들 (다국어 무료 지원)
```

- 생성 언어는 SONAR 디코더만 바꾸면 됨 → **재학습 없이 언어 전환 가능**

---

### 9. 성능 평가

**XLSum (다국어 요약) 제로샷**: 7B LCM이 Llama-3.1-8B를 능가

| 항목 | Llama-3.1-8B | LCM (Two-Tower, 7B) |
|------|-------------|---------------------|
| ROUGE-L (요약) | 낮음 | **33.64 (SOTA급)** |
| 반복률 (REP-4) | 상대적으로 높음 | **낮음** |
| 유창성 (CoLA) | 높음 | 약간 낮음 |
| 추상성 | 추출적 경향 | **더 추상적** |
| 긴 문서(5K+ 단어) | 품질 저하 | **안정적** |

- 명시적 학습 없이도 **다국어 요약** 성능이 우수 → 언어 독립성의 증거

### 10. 한계와 제약

| 구분 | 내용 |
|------|------|
| 표현 해상도 | 문장 단위라 단어 수준 섬세함 상실 (예: 고유명사, 숫자) |
| SONAR 의존성 | 동결된 SONAR의 표현력이 전체 상한선을 결정 |
| 다중 모드 | Diffusion 비용이 크고 수렴 어려움 |
| 비텍스트 | 이미지·비디오는 아직 실험적 |
| 평가 지표 | 임베딩 단위 생성을 평가할 표준 메트릭 부재 |
| 긴 문맥 학습 | 문서 수준 병렬 데이터 부족 |

## 내가 얻은 인사이트

### 설계 관점

1. **"Tokenizer 종속성 탈출"이 진짜 혁신**
   - BPE/SentencePiece가 고정되면 다국어·멀티모달 통합에 한계가 생긴다.
   - SONAR처럼 **언어·모달리티 독립 공간**을 주축으로 두면, 입력/출력 모듈만 교체해 새 도메인을 흡수할 수 있다.
   - 이는 어댑터·LoRA보다 더 본질적인 "모듈화" 접근이다.

2. **계층적 추론(Hierarchical Reasoning)의 전조**
   - 인간은 단어 → 문장 → 단락 → 아이디어 → 계획으로 계층적으로 사고한다.
   - LCM은 "문장"을 기본 단위로 잡았지만, 이후 "단락", "에피소드" 단위로 쌓는 multi-level LCM이 자연스러운 다음 수순이다.
   - Chain-of-Thought의 시퀀스 길이 문제를 근본적으로 해결할 수 있다.

### 시스템/엔지니어링 관점

3. **컨텍스트 윈도우 경제성이 완전히 달라진다**
   - 1,000 토큰 ≈ 50 개념. attention 비용 O(N²)에서 N이 20배 줄면 실효 비용은 **400배** 절감.
   - 긴 문서 RAG, 장시간 대화 기억 등에서 구조적 우위.

4. **Two-Tower 구조가 주는 교훈**
   - "맥락 인코딩(1회) + 반복 디노이징(T회)"으로 비용을 분리하는 패턴은 Diffusion 기반 LLM 일반에 적용 가능한 **일반 원리**이다.
   - Stable Diffusion의 cross-attention 구조와 유사한 발상 → 텍스트 생성에 diffusion이 들어올 때 동일한 패턴이 주류가 될 가능성.

### 트레이드오프 관점

5. **표현 해상도 vs 시퀀스 길이의 맞교환**
   - 문장 단위 추론은 길이를 줄이지만, "정확한 숫자/이름"을 잃기 쉽다.
   - 실제 서비스에서는 **Hybrid** (LCM으로 구조/흐름 결정 → LLM으로 문장 디테일 생성)가 현실적 설계일 수 있다.

6. **동결(frozen) SONAR의 양날의 검**
   - 학습은 쉬워지지만 SONAR의 한계를 그대로 물려받는다.
   - 도메인 특화(의료·법률)에서는 "도메인 SONAR"를 따로 학습해야 실용적.

### 하이프/현실 관점

7. **"LLM의 종말"이 아니라 "LLM의 역할 재정의"**
   - LCM은 LLM을 대체하기보다 **상위 계획자**로 자리 잡을 가능성이 높다.
   - LLM: 문장 내부 단어 생성 / LCM: 문장들의 흐름 계획 → **2-tier 언어 모델 스택**이 표준화될 수 있다.

8. **"AGI 빠진 퍼즐"에 가까운 방향성**
   - 현재 LLM이 잘 못하는 장기 계획, 일관성, 언어 독립 추론을 정면으로 겨냥.
   - 다만 학습 데이터·평가 인프라 부재가 병목 → **인프라 투자(병렬 문서 데이터, 의미 기반 메트릭)** 가 실제 성패를 가를 것이다.
